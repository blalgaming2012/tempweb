# وثيقة متطلبات تطبيق Tempweb

## 1. اسم التطبيق
Tempweb

## 2. أيقونة التطبيق
- استخدام الأيقونة favicon.png كأيقونة التطبيق بدون أي تأثيرات بصرية (بدون توهج أو بطاقات أو ظلال)
- عرض الأيقونة بشكل نظيف ومباشر في المتصفح وعلامات التبويب وداخل واجهة التطبيق

## 3. وصف التطبيق
تطبيق ويب متكامل وتفاعلي لإدارة خدمة العملاء والعمال، مبني على حزمة تقنيات حديثة (React/Vite، Tailwind CSS، Supabase) مع تركيز على تجربة المستخدم الأنيقة والأمان العالي والحركات المتحركة.

## 4. الوظائف الرئيسية
\n### 4.1 لوحة تحكم خدمة العملاء\n- واجهة مخصصة لإدارة طلبات العملاء\n- عرض وإدارة بيانات الخدمات
- متابعة حالة الطلبات
\n### 4.2 لوحة تحكم العمال
- واجهة مخصصة للعمال لعرض المهام المخصصة لهم فقط
- تحديث حالة المهام
- الوصول المحدود للبيانات الخاصة بكل عامل

### 4.3 نظام الأمان
- تطبيق Row Level Security (RLS) في Supabase لضمان عدم وصول العمال إلا لبياناتهم الخاصة
- استعلامات قاعدة بيانات محسّنة لتحقيق أقصى سرعة أداء
\n### 4.4 إدارة المستخدمين
- نظام تسجيل دخول وصلاحيات متعدد المستويات
- حفظ تفضيلات المستخدم (الثيم) في localStorage
\n### 4.5 دعم اللغات المتعددة
- دعم كامل لجميع اللغات مع إمكانية التبديل بينها
- واجهة متعددة اللغات تتكيف مع اختيار المستخدم
- حفظ تفضيل اللغة للمستخدم
\n## 5. المتطلبات التقنية\n
### 5.1 حزمة التقنيات
- Frontend: React مع Vite
- Styling: Tailwind CSS
- Backend/Database: Supabase
- Internationalization: مكتبة i18n لدعم اللغات المتعددة
\n### 5.2 قاعدة البيانات
- استخدام Supabase كقاعدة بيانات قوية ومؤمنة
- تطبيق سياسات أمان صارمة (Row Level Security)
- تشفير البيانات الحساسة
- نسخ احتياطي تلقائي للبيانات
- مراقبة الأداء والاستعلامات

### 5.3 الخطوط\n- الخط الأساسي: Cairo (font-sans)
- خط العناوين: Montserrat (font-heading)
\n### 5.4 الأمان
- تطبيق Row Level Security بصرامة في Supabase
- تحسين جميع استعلامات قاعدة البيانات
\n### 5.5 دعم اللغات
- تطبيق نظام ترجمة شامل يدعم جميع اللغات
- إمكانية إضافة لغات جديدة بسهولة
- تخزين ملفات الترجمة بشكل منظم

## 6. أسلوب التصميم

### 6.1 نظام الألوان
- دعم كامل للوضع الداكن والساطع (Dark/Light Mode)
- زر تبديل الثيم في شريط التنقل مع حفظ التفضيل
- ألوان متناسقة مع كلا الوضعين

### 6.2 التفاصيل البصرية والحركية
- تأثير الخلفية المتفاعلة مع التمرير (Parallax Effect) باستخدام الكلاس .parallax-bg
- تصميم أنيق ومتسق عبر جميع المكونات
- انتقالات سلسة بين الوضع الداكن والساطع
- حركات متحركة (Animations) عند تحميل العناصر والتفاعل معها
- تأثيرات Hover نشطة على الأزرار والروابط
- انتقالات سلسة بين الصفحات (Page Transitions)
- تحريك العناصر عند الظهور (Fade-in/Slide-in Effects)

### 6.3 التخطيط العام
- واجهة نظيفة ومنظمة
- تسلسل منطقي للعناصر
- تجربة مستخدم سلسة ومتكاملة
- واجهة نشطة ومتفاعلة مع استجابة فورية للمستخدم
